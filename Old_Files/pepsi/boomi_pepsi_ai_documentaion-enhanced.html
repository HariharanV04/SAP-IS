API Overview

This integration process extracts employee demographic data from SuccessFactors Employee Central and delivers it to the NAVEX vendor platform for compliance and ethics management purposes. The system supports both full and delta extraction modes with configurable filtering capabilities.

Key Features: - Automated employee data extraction from SuccessFactors Employee Central - Dynamic filtering based on employee class, legal entity, territory codes, and employment status - Delta processing with last successful run date tracking - Secure file delivery to NAVEX vendor via SFTP with PGP encryption - Archive management with PepsiCo internal SFTP storage - Comprehensive error handling and notification system - Control reporting and audit trail functionality

Authentication: Process property-based configuration with PGP encryption for secure data transmission

Data Processing Modes: - Delta Mode: Incremental extraction based on last modified date - Full Mode: Complete dataset extraction with configurable date range

File Formats: CSV output format with PGP encryption for secure vendor delivery

Current Dell Boomi Flow Logic

Process Flow Overview

The Employee Central to NAVEX integration is a comprehensive data extraction and delivery system that:

  1. Initializes with error reporting properties and dynamic filtering configuration
  2. Determines processing mode (Delta vs Full) based on last successful run date
  3. Builds dynamic WHERE clauses for targeted employee data extraction
  4. Extracts employee demographics from SuccessFactors Employee Central
  5. Transforms data to NAVEX-required CSV format with field mappings
  6. Delivers files securely to NAVEX vendor SFTP with PGP encryption
  7. Archives files to PepsiCo internal SFTP repository
  8. Generates control reports and sends success/error notifications

The process includes sophisticated error handling, parallel processing capabilities, and configurable filtering to support various business scenarios including legal entity separation, territory-based filtering, and employee class restrictions.

Component Type: Start Event

Purpose: Initiates the employee data extraction process with decision logic for processing mode determination

Configuration Details: - Process Name: EC372 - EC to NAVEX (OB) - Version: 75 - Trigger Type: Scheduled or manual execution - Initial Processing: Delta vs Full run determination

Component Type: Decision

Purpose: Determines whether to perform incremental (delta) or full data extraction based on last successful run date

Configuration Details: - Decision Criteria: Filter_LSRD property evaluation - Logic: equals comparison - Branch Logic: Routes to different property setting paths

Data Flow: - Input: Process execution context - Processing: Evaluates Filter_LSRD (Last Successful Run Date) property - Output: Routes to appropriate date range setting logic

Component Type: Decision

Purpose: Business decision to check if Filter LSRD (Last Successful Run Date) is NULL

Configuration Details: - Decision Name: Filter LSRD is NULL - Logic: equals comparison - Purpose: Determines initial setup vs ongoing delta processing
